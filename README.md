# üè¥ CTF Framework

A production-grade **Capture The Flag challenge platform** built in Python ‚Äî featuring challenge scaffolding, flag validation with rate limiting, dynamic scoring, Docker deployment, a REST API, and a hacker-aesthetic web interface.

---

## üöÄ Quick Start
```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Initialize a new CTF event
python ctf.py init

# 3. Create some challenges
python ctf.py create web "SQL Injection 101" --points 100 --difficulty easy
python ctf.py create pwn "Buffer Overflow" --points 300 --difficulty hard
python ctf.py create crypto "Broken RSA" --points 200 --difficulty medium

# 4. Start the web server
python ctf.py server --port 5000
```

Visit `http://localhost:5000` for the web interface.

## üéÆ CLI Reference
```bash
# Create a challenge
python ctf.py create <category> "<name>" [--points N] [--difficulty easy|medium|hard|insane]

# Categories: web, pwn, crypto, forensics, reverse, misc, osint, network, mobile, blockchain

# List challenges
python ctf.py list
python ctf.py list --category web
python ctf.py list --difficulty hard

# Submit a flag
python ctf.py submit <challenge_id> "CTF{...}" --team myteam

# Scoreboard
python ctf.py scoreboard --top 20

# Start server
python ctf.py server --host 0.0.0.0 --port 8080 --debug

# Docker
python ctf.py docker build <challenge_id>
python ctf.py docker deploy <challenge_id> --port 9001
```

## üåê REST API

| Method | Endpoint | Description |
|--------|----------|-------------|

## üîê Security Features

- **Rate Limiting** ‚Äî Token-bucket (10 attempts / 60s per team per challenge)
- **Timing Attack Protection** ‚Äî SHA-256 constant-time flag comparison
- **First Blood Tracking** ‚Äî First solve per challenge recorded
- **Audit Log** ‚Äî All submissions logged with timestamps & IPs
- **Dynamic Scoring** ‚Äî Points decay as more teams solve
- **Docker Hardening** ‚Äî `--read-only`, `no-new-privileges`, tmpfs
